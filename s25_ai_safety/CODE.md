# s25 AI 安全与对齐 -- 代码说明与运行报告

## 程序做了什么
演示 AI 安全的四个关键议题：幻觉检测（基于内置知识库检索验证模型生成的事实性声明）、越狱攻击检测（正则匹配 + 关键词过滤常见越狱提示词模式如 DAN/角色扮演/编码绕过）、偏见分析（对预定义问答对按职业/性别等维度做统计偏差度量）、内容安全评估（多维度安全评分：暴力/色情/仇恨/歧视/自残）。所有测试运行在模拟问答对上，输出结构化安全报告。

## 运行方法
```bash
cd s25_ai_safety/code
python demo.py
```

## 运行结果

### 输出摘要（命令行文本，无生成图片）
- 幻觉检测报告：每条声明的判定结果（事实一致/不一致/不确定）、置信度分数、解释说明
- 越狱攻击测试：每个攻击提示的检测结果（通过/拦截）、安全评分、危险模式识别详情
- 偏见统计分析：不同职业/性别等维度的回答分布统计、偏差度量指标
- 内容安全多维度评分：5 个维度的得分（0-10 分），含详细扣分项说明
- 综合安全总结：整体安全率、检测到的各类问题数量及严重程度汇总

## 代码结构
- `class HallucinationDetector` -- 幻觉检测器：内置 10 条事实知识库，基于关键词重叠 + 知识库匹配判断一致性
  - `check_factuality()` -- 对输入声明逐条验证，返回事实性判定
  - `_tokenize()` -- 中文分词辅助
- `class JailbreakDetector` -- 越狱检测器：内置越狱模式库（DAN/角色扮演/编码绕过等）
  - `detect()` -- 正则匹配 + 关键词检测识别越狱企图
  - `sanitize()` -- 输入净化，移除可疑模式
- `class BiasAnalyzer` -- 偏见分析器：预定义测试用例（含性别/职业/年龄维度的问答对）
  - `analyze()` -- 对各组输出做统计分析和偏差度量
- `class ContentSafetyEvaluator` -- 内容安全评估：5 维度安全评分
  - `evaluate()` -- 基于关键词和规则的多维度打分（暴力/色情/仇恨/歧视/自残）
- `run_hallucination_demo()` -- 幻觉检测演示：展示正例（正确事实）和负例（虚构事实）的检测效果
- `run_jailbreak_demo()` -- 越狱攻击测试演示：展示各种攻击模式及其检测结果
- `run_bias_demo()` -- 偏见测试演示：展示不同维度下的统计偏差
- `run_safety_demo()` -- 内容安全评估演示：展示多维度安全评分体系
- `main()` -- 主流程：依序运行四个安全模块演示

## 运行环境
- Python 依赖: numpy（基础），scikit-learn（可选，用于相似度计算）
- 硬件需求: CPU 即可（纯规则 + 正则匹配，极小内存 ~10MB）
- 预计运行时间: ~5-10 秒
- 注：本章为概念性演示，使用规则匹配和内置知识库，不涉及真实 LLM 调用
